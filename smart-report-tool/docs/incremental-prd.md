# 智能报告生成工具 - 增量需求文档 (v1.1)

## 1. 需求概述

在现有系统（v1.0）基础上，扩展三大核心模块：
1. 用户注册与审批流程
2. 脚本上传与管理增强
3. 三级角色权限体系重构

## 2. 用户故事

### 新用户视角
- 作为新用户，我希望通过注册页面自主申请账号，以便加入平台使用报告生成功能。
- 作为新用户，我希望注册后能收到账户审核状态的反馈，以便了解何时可以登录。

### 高级成员视角
- 作为高级成员，我希望上传并管理自己的巡检脚本（编辑、删除、执行），以便团队共享自动化能力。
- 作为高级成员，我希望能够删除自己不再需要的历史报告，以便保持工作区整洁。

### 管理员视角
- 作为管理员，我希望审批或拒绝新用户注册申请，以便控制平台访问权限。
- 作为管理员，我希望查看所有脚本的执行记录，以便审计和排查问题。

## 5. 数据模型变更

### 5.1 users 表（变更）

```ts
interface User {
  id: string;

  role: 'admin' | 'senior' | 'member';
  displayName: string;
  status: 'pending' | 'active' | 'rejected';  // 新增

```

### 5.2 scripts 表（变更）

```ts
interface Script {
  id: string;
  name: string;
  description: string;           // 新增：备注说明
  scriptType: 'python' | 'bat' | 'ps1' | 'sh' | 'powershell';  // 新增
  version: string;               // 新增：版本号

  fileName: string;
  fileSize: number;
  content: string;
  uploadedAt: string;
  uploadedBy: string;
}
```

### 5.3 execution_logs 表（新增）

```ts
interface ExecutionLog {
  id: string;
  scriptId: string;
  scriptName: string;
  executedBy: string;
  executedById: string;
  targetHost?: string;
  status: 'pending' | 'running' | 'success' | 'failed';
  output: string[];
  startedAt: string;
  completedAt?: string;
}
```

## 7. 核心业务流程

### 7.1 用户注册审批流程

```
[用户访问注册页] → [填写注册表单] → [前端校验通过]
  → [提交注册] → [创建用户，status=pending]
  → [提示"等待管理员审核"]
  → [管理员查看待审核列表]
  → [批准 → status=active / 拒绝 → status=rejected]
```

### 7.2 脚本执行流程（前端模拟）

```
[用户点击执行] → [弹出执行配置对话框]
  → [用户填写目标主机等参数]
  → [点击确认执行]
  → [创建 execution_log，status=running]
  → [模拟逐行输出日志]
  → [执行完成 → status=success/failed]
  → [展示执行结果]
```
