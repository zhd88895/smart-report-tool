## 2. 产品定义

### 2.1 Product Goals

1. **提升运维巡检效率**：通过脚本化、模板化、AI 辅助，将传统手动报告编制时间缩短 70% 以上。
2. **降低报告生成门槛**：提供可视化四步向导，非技术人员也能独立完成专业巡检报告。
3. **构建可检索的知识资产**：历史报告与对话记录结构化沉淀，支持自然语言检索与智能分析。

### 2.2 User Stories

**管理员视角**
- 作为管理员，我希望统一管理所有巡检脚本和用户账号，以便控制平台使用范围与数据安全。
- 作为管理员，我希望查看全量 AI 对话记录，以便审计和优化助手回答质量。

**普通用户视角**
- 作为普通用户，我希望按分类上传巡检脚本和日志，并通过四步向导快速生成报告，以便减少重复劳动。
- 作为普通用户，我希望与 AI 助手对话来分析报告或检索历史报告，以便快速获取运维洞察。

**只读用户视角**
- 作为只读用户，我希望在报告下载中心查看和下载已生成的报告，以便了解系统巡检结果而不参与生成过程。

## 5. 核心业务流程

### 5.1 报告生成四步流程

```
[步骤① 日志上传] ──→ [步骤② 填写报告信息] ──→ [步骤③ 模板选择与预览] ──→ [步骤④ 提交生成]
```

#### 步骤① 日志上传

1. 用户进入报告生成页，默认展示步骤①。
2. 页面顶部显示五类日志类型标签页：**主机 / 存储 / 数据库 / 虚拟化 / 网络**。
3. 切换标签页后，动态展示对应类型的上传区域。
4. 上传区域支持 **点击选择文件** 和 **拖拽上传**。
5. 已上传文件以列表展示，显示文件名、大小，支持删除。
6. 若选择「数据库」类型，额外显示「调用 AI Agent 分析」可选开关。
7. 点击「下一步」进入步骤②。

#### 步骤② 填写报告信息

1. **用户名称**：自动填充当前登录用户，不可编辑。
2. **报告名称**：必填，文本输入，长度限制 2-100 字符。
3. **报告日期**：日期选择器，默认系统当前时间，允许修改。
4. **作者**：自动填充当前登录用户真实姓名/账号，不可编辑。
5. 点击「上一步」返回步骤①，点击「下一步」进入步骤③。

#### 步骤③ 模板选择与预览

1. 展示内置模板网格卡片，每张卡片显示模板名称、缩略图、适用场景。
2. 点击模板卡片即选中（高亮边框），同时右侧/下方展示模板预览。
3. 支持「自定义模板上传」入口，上传后可立即选用。
4. 点击「上一步」返回步骤②，点击「下一步」进入步骤④。

#### 步骤④ 提交生成并展示进度

1. 汇总展示前三步信息：日志文件清单、报告信息、所选模板。
2. 点击「提交生成」后，进入生成中状态。
3. 展示进度条（模拟/真实进度）与当前处理阶段文案。
4. 生成完成后，状态变为「成功」，显示「下载报告」和「返回列表」按钮。
5. 若生成失败，展示错误原因与「重试」按钮。

## 7. UI 设计规范

### 7.1 色彩体系

主色调采用 **蓝 / 红 / 绿** 三色，分别对应信息/操作、告警/强调、成功/安全语义：

| 语义 | 色值 | 使用场景 |
|------|------|----------|
| 主色蓝 | `#2563EB` | 导航栏背景、主要按钮、链接、选中状态 |
| 深蓝 | `#1E40AF` |  hover 状态、激活菜单项 |
| 强调红 | `#DC2626` | 危险操作（删除）、告警提示、必填标记 |
| 浅红 | `#FEE2E2` | 错误提示背景 |
| 成功绿 | `#16A34A` | 成功状态、进度完成、通过标识 |
| 浅绿 | `#DCFCE7` | 成功提示背景 |
| 背景灰 | `#F8FAFC` | 页面底色 |
| 卡片白 | `#FFFFFF` | 卡片、弹窗、表单区域 |
| 文字主色 | `#1E293B` | 正文标题 |
| 文字次色 | `#64748B` | 辅助说明、占位文字 |

### 7.2 布局规范

- **顶部导航栏**：固定高度 `64px`，左侧 Logo + 产品名，右侧用户头像下拉菜单（个人设置/退出）。
- **侧边菜单栏**：宽度 `240px`（可折叠为 `64px`），图标 + 文字菜单项，当前页高亮。
- **主内容区**：`padding: 24px`，内容最大宽度 `1440px` 居中，自动适应剩余宽度。
- **响应式断点**：
  - PC：`≥1280px`，展开侧边栏
  - 平板：`768px–1279px`，侧边栏可折叠，内容区自适应
  - 手机：`<768px`，本阶段不做重点适配，保证基础可用即可

### 7.3 组件风格

- 使用 **shadcn/ui** 组件库，保持原生 Tailwind 风格。
- 按钮：主按钮（蓝底白字）、次按钮（白底蓝边）、危险按钮（红底白字）。
- 表单输入：圆角 `8px`，聚焦状态蓝色边框。
- 卡片：白色背景，圆角 `12px`，阴影 `shadow-sm`，hover 时 `shadow-md` 过渡。
- 表格：表头浅灰背景，行 hover 浅蓝背景，操作按钮集中右侧。
- 步骤条（Stepper）：四步流程使用横向步骤条，已完成/当前/待完成三色区分。

## 8. 数据模型（前端存储设计）

基于 localStorage / IndexedDB 的纯前端存储方案：

### 8.1 用户表 (users)
```ts
interface User {
  id: string;           // 唯一标识
  username: string;     // 登录账号
  password: string;     // 加密存储（简易 hash）
  role: 'admin' | 'user' | 'readonly';
  displayName: string;  // 显示名称
  createdAt: string;    // ISO 日期
}
```

### 8.2 脚本表 (scripts)
```ts
interface Script {
  id: string;
  name: string;
  category: 'host' | 'storage' | 'database' | 'virtualization' | 'network';
  fileName: string;
  fileSize: number;
  content: string;      // base64 或文本内容
  uploadedAt: string;
  uploadedBy: string;   // user id
}
```

### 8.3 报告表 (reports)
```ts
interface Report {
  id: string;
  name: string;
  type: 'host' | 'storage' | 'database' | 'virtualization' | 'network';
  date: string;
  author: string;
  authorId: string;
  templateId: string;
  status: 'generating' | 'success' | 'failed';
  logs: string[];       // 关联日志文件名列表
  fileUrl?: string;     // 生成后报告文件（Blob URL 或 base64）
  createdAt: string;
}
```

### 8.4 对话记录表 (conversations)
```ts
interface Conversation {
  id: string;
  userId: string;
  userName: string;
  messages: Array<{
    role: 'user' | 'assistant';
    content: string;
    intent?: 'query_report' | 'analyze_data' | 'general';
    timestamp: string;
  }>;
  createdAt: string;
  updatedAt: string;
}
```

## 9. AI 助手交互设计

### 9.1 界面形态

- **独立页面** (`/assistant`)：全屏对话界面，左侧历史会话列表，右侧聊天区域。
- **快捷入口**：在报告生成结果页和报告下载页提供「询问 AI」浮窗按钮，快速唤起侧边对话面板。

### 9.2 功能交互

1. **多轮对话**：支持上下文关联的连续对话。
2. **意图识别**：用户发送消息后，前端先通过关键词/规则初判意图类型：
   - 包含「报告」「查找」「搜索」→ `query_report`
   - 包含「分析」「解读」「性能」→ `analyze_data`
   - 其他 → `general`
3. **巡检报告智能分析**：用户在报告下载页选择报告后点击「AI 分析」，携带报告 ID 打开助手，AI 读取报告内容进行解读。
4. **巡检报告检索**：用户输入自然语言查询，AI 检索 `reports` 表并返回匹配结果。
5. **对话历史**：每次对话自动保存到 `conversations` 表，用户可查看历史会话列表。
6. **管理员审计**：管理员在「对话记录」页查看全部用户的对话历史，支持按用户、时间筛选。
